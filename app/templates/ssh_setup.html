SSH Auto-Wake Setup

Automatically wake a sleeping machine when you SSH into it — from any terminal, VS Code Remote-SSH, Windsurf, Cursor, or JetBrains Gateway.

{# ── How it works ─────────────────────────────────────────────────────── #}

How it works

  1. SSH runs a small local script via ProxyCommand before connecting.
  2. The script calls this API — if the machine is already online it exits immediately.
  3. If the machine is sleeping, it sends the wake packet and waits until TCP/SSH responds.
  4. SSH then connects normally to the now-awake machine.

Step 2 — Install the wol-wake script

Run this once on the machine you SSH from (your laptop/desktop).

{# OS tab switcher #}
{# Linux/macOS bash script #}

Requires curl and jq (sudo apt install curl jq / brew install curl jq). 

mkdir -p ~/.config/wol-monkey
read -rsp 'Paste your WoL-Monkey token: ' _TOKEN && echo
echo "$_TOKEN" > ~/.config/wol-monkey/token
unset _TOKEN
chmod 600 ~/.config/wol-monkey/token

install -Dm 755 /dev/stdin ~/.local/bin/wol-wake << 'EOF'
#!/usr/bin/env bash
# wol-wake — auto-wake via WoL-Monkey before SSH connects
set -euo pipefail
MACHINE_ID="${1:?machine-id required}"
SSH_HOST="${2:?ssh-host required}"
SSH_PORT="${3:-22}"
WOL_BASE="${WOL_MONKEY_URL:-{{ base_url }}}"
TOKEN_FILE="${WOL_MONKEY_TOKEN_FILE:-$HOME/.config/wol-monkey/token}"
TOKEN="${WOL_MONKEY_TOKEN:-$(cat "$TOKEN_FILE" 2>/dev/null)}"
POLL_TIMEOUT="${WOL_MONKEY_TIMEOUT:-90}"
if [[ -z "$TOKEN" ]]; then
  echo "wol-wake: no token — set WOL_MONKEY_TOKEN or create $TOKEN_FILE" >&2; exit 1
fi
AUTH=(-H "Authorization: Bearer $TOKEN")
STATE=$(curl -sf "${AUTH[@]}" \
  "$WOL_BASE/api/machines/$MACHINE_ID/status" | jq -r '.state' 2>/dev/null || echo "unknown")
if [[ "$STATE" != "online" ]]; then
  echo "wol-wake: $STATE — waking..." >&2
  curl -sf -X POST "${AUTH[@]}" -H "Content-Type: application/json" \
    -d "{\"ensure_online\":true,\"poll_timeout_s\":$POLL_TIMEOUT}" \
    "$WOL_BASE/api/machines/$MACHINE_ID/wake/direct" >/dev/null || true
  echo "wol-wake: connecting" >&2
fi
exec nc "$SSH_HOST" "$SSH_PORT"
EOF

echo "Installed. Make sure ~/.local/bin is on your PATH."

{# Windows PowerShell #}

For Windows OpenSSH without WSL. Paste this once into a PowerShell terminal — it will prompt for your token and write wol-wake.ps1 to your profile folder automatically. 

# Step 1 — store your token
New-Item -ItemType Directory -Force "$env:APPDATA\wol-monkey" | Out-Null
$_token = Read-Host -AsSecureString 'Paste your WoL-Monkey token'
[System.Runtime.InteropServices.Marshal]::PtrToStringAuto(
    [System.Runtime.InteropServices.Marshal]::SecureStringToBSTR($_token)
) | Set-Content "$env:APPDATA\wol-monkey\token" -NoNewline
Remove-Variable _token

# Step 2 — write the wol-wake.ps1 script
@'
param(
    [Parameter(Mandatory)][string]$MachineId,
    [Parameter(Mandatory)][string]$SshHost,
    [string]$SshPort = "22"
)
$WolBase   = if ($env:WOL_MONKEY_URL)        { $env:WOL_MONKEY_URL }        else { "{{ base_url }}" }
$TokenFile = if ($env:WOL_MONKEY_TOKEN_FILE) { $env:WOL_MONKEY_TOKEN_FILE } else { "$env:APPDATA\wol-monkey\token" }
$Token     = if ($env:WOL_MONKEY_TOKEN)      { $env:WOL_MONKEY_TOKEN }      else { (Get-Content $TokenFile -EA SilentlyContinue) }
$Timeout   = if ($env:WOL_MONKEY_TIMEOUT)    { $env:WOL_MONKEY_TIMEOUT }    else { 90 }
if (-not $Token) { Write-Error "wol-wake: no token"; exit 1 }
$H = @{ Authorization = "Bearer $Token" }
try {
    $s = Invoke-RestMethod "$WolBase/api/machines/$MachineId/status" -Headers $H
    if ($s.state -ne "online") {
        Write-Host "wol-wake: $($s.state) — waking..." -ForegroundColor Yellow
        $b = "{`"ensure_online`":true,`"poll_timeout_s`":$Timeout}"
        Invoke-RestMethod "$WolBase/api/machines/$MachineId/wake/direct" `
            -Method Post -Headers $H -ContentType "application/json" -Body $b | Out-Null
        Write-Host "wol-wake: online — connecting" -ForegroundColor Green
    }
} catch { Write-Warning "wol-wake: API error — connecting anyway" }
& "$env:SystemRoot\System32\OpenSSH\ssh.exe" -W "${SshHost}:${SshPort}" localhost
'@ | Set-Content "$env:USERPROFILE\wol-wake.ps1" -Encoding UTF8

Write-Host "Installed to $env:USERPROFILE\wol-wake.ps1" -ForegroundColor Green

Then in Step 3, use ProxyCommand powershell -File %USERPROFILE%\wol-wake.ps1 <machine-uuid> %h %p in your SSH config.

Step 3 — Add to your SSH config

Add the relevant block to your SSH config on your client machine. The tab syncs with your Step 2 selection.

# ~/.ssh/config — {{ m.name }}
Host {{ m.name | lower | replace(' ', '-') }}
    HostName {{ m.hostname or m.ip_address }}
    User YOUR_USERNAME
    ProxyCommand wol-wake {{ m.id }} %h %p
    ServerAliveInterval 30
    ServerAliveCountMax 3

{% endfor %}

{# ── Notes ─────────────────────────────────────────────────────────────── #}

Notes

  • VS Code / Windsurf / Cursor / JetBrains Gateway — these IDEs read ~/.ssh/config directly, so Step 3 is all you need after the script is installed.
  • PATH — if wol-wake: command not found, add export PATH="$HOME/.local/bin:$PATH" to your ~/.bashrc or ~/.zshrc.
  • Tailscale — set WOL_MONKEY_URL to your Tailscale address in the ProxyCommand: ProxyCommand env WOL_MONKEY_URL=http://<wol-monkey-tailscale-ip> wol-wake <id> %h %p
  • Token security — the token file is stored at ~/.config/wol-monkey/token with mode 600 (readable only by you). You can revoke it at any time from API Tokens.
